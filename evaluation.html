Rome.ista Photography Page

by Luke Rebeiro

Posted on September 19th 2020

Brief

I set out to build a website for a photographer who wanted to display their work. The client wanted an interactive website featuring an introduction page, gallery section and a page where people could contact them.

Design

I started by drawing my ideas onto pieces of paper which would help me with wireframing at a later stage. As you can see from the images below. I have included a Home page, About Me page, Gallery page, Contact page and a contents’ sections. Each page includes a responsive navigation bar which responds to mobile, tablets and desktop devices. As shown from my drawings I have placed images and text on each page to bring together the appearance and decide on the layout needed to adhere to the clients brief and made it responsive.

Wireframing was the next stage in the design process. I used Adobe XD to design a wireframe for the website. As you can see from the images below, I have designed various layouts for the website which include, navigation bars, footer bars, logo placement and social media icon placement. Each web page has the same navigation and footer placement on the page and this was to ensure the design was consistent throughout the project.

After completing the wire-framing process, I moved onto designing the final draft using the finalised wireframes. I inserted images and colours provided by the client. I used Adobe Photoshop and Illustrator to design the logo for the client.

Coding

I used Visual Studio Code as my main coding software. I have used HTML, CSS and Javascript coding languages in this project as you can see from the images below.

After writing the full code for the website, I reviewed and changed the look of the navigation bar because it was not user friendly. I also moved the logo from the centre of the navigation bar to the left and the links for the other pages to the right.


The gallery section needed to be adjusted. As you can see from the original design, all the pictures are close together on one page. This needed to be changed because it was not responsive enough for the idea I was trying to create. I drew a new gallery web page on paper where the photos were split apart and when the user clicked onto the photo it would appear in a light-box, which enlarged the photos to their original size. This worked on mobiles, tablets and desktops therefore making the gallery section responsive which kept to the client brief. I also changed the design of the contact section because it was very basic and did not match the design of the rest of the site.

Conclusion

To conclude, the website I designed for my client adhered to the brief that was given. The website was challenging in some areas particularly the gallery section. To overcome this I upskilled in interactive designs to create a user friendly, efficient page for my client’s work to be displayed. Rome.Ista Photography is a fully functioning interactive website successfully displaying my client’s work.
